# 智能办公桌面助手集群（Smart Desktop Assistant Cluster）

## 一、场景背景与核心痛点

### 日常工作学习场景
- **高频痛点**：
  1. **信息散落**：文件、笔记、任务分散在不同设备和平台，查找困难
  2. **专注度低**：频繁的通知和上下文切换，导致注意力分散
  3. **效率低下**：重复性任务多，如整理文件、记录笔记、设置提醒
  4. **健康问题**：久坐、不良姿势、眼部疲劳等办公常见健康问题
  5. **协作不畅**：团队成员之间的信息共享和任务协调效率低

### 目标用户
- **办公室职员**：需要高效管理日常工作任务和信息
- **学生**：需要整理学习资料、管理学习计划
- **自由职业者**：需要自我管理工作流程和时间


## 二、智能办公桌面助手集群设计

### 硬件集群配置
- **中央控制终端**：笔记本电脑/平板（现有设备）
- **桌面节点**：
  1. **智能台灯**：调节光线、显示时间和状态
  2. **智能音箱**：语音交互、播放提示音
  3. **智能显示器**：显示任务、提醒和信息卡片
  4. **智能按钮**：物理触发不同功能
  5. **智能传感器**：监测环境（光线、温度、湿度）和用户状态（坐姿、专注度）
- **扩展设备**：
  1. **智能杯垫**：监测饮水量和水温
  2. **智能植物**：显示空气质量、增加桌面生机

### 硬件布局
- **核心区域**：桌面中央，放置智能显示器和智能台灯
- **交互区域**：桌面右侧，放置智能按钮和智能传感器
- **辅助区域**：桌面左侧，放置智能音箱和智能杯垫
- **边缘区域**：桌面角落，放置智能植物

### 硬件实现方案（48h可完成）
- **中央终端**：使用现有笔记本电脑/平板
- **桌面节点**：
  - ESP32 + LED矩阵（智能台灯）
  - ESP32 + 扬声器（智能音箱）
  - ESP32 + 小型显示屏（智能显示器）
  - ESP32 + 按钮（智能按钮）
  - ESP32 + 传感器模块（智能传感器）
- **扩展设备**：
  - ESP32 + 水位传感器（智能杯垫）
  - ESP32 + 空气质量传感器（智能植物）
- **连接方式**：所有节点通过Wi-Fi/BLE连接到中央终端

## 三、核心功能与多模态交互

### 1. 智能信息管理
- **触发**：
  - 语音指令："桌面助手，整理我的文件"
  - 物理触发：点击智能按钮（短按）
  - 自动触发：每天特定时间自动整理

- **集群效果**：
  - **视觉**：智能显示器显示文件分类界面，智能台灯变为蓝色脉冲光效
  - **听觉**：智能音箱播放"开始整理文件"的提示音，整理完成后播放成功音效
  - **触觉**：智能按钮轻微振动反馈

- **功能**：
  - 自动扫描桌面文件，按类型（文档、图片、视频、压缩包）分类
  - 智能识别文件内容，按项目或主题进一步细分
  - 生成整理报告，显示整理前后的对比
  - 提供文件搜索功能，快速定位文件

### 2. 专注模式
- **触发**：
  - 语音指令："桌面助手，开启专注模式"
  - 物理触发：长按智能按钮（3秒）
  - 应用集成：从专注应用（如Forest）触发

- **集群效果**：
  - **视觉**：智能台灯变为暖黄色，亮度调暗至70%，智能显示器显示专注倒计时和目标
  - **听觉**：智能音箱播放白噪音或专注音乐，屏蔽其他通知声音
  - **环境**：智能传感器监测环境光线，自动调节台灯亮度

- **功能**：
  - 屏蔽电脑通知，避免干扰
  - 设置专注时间（默认25分钟），可自定义
  - 检测用户专注度，通过键盘鼠标活动判断
  - 达到时间后，智能台灯变为绿色，播放休息提示音
  - 提供休息建议，如眼部放松、伸展运动

### 3. 健康助手
- **触发**：
  - 语音指令："桌面助手，检查我的健康状态"
  - 自动触发：智能传感器每小时自动监测
  - 定时提醒：每天固定时间（如10点、14点、16点）

- **集群效果**：
  - **视觉**：智能显示器显示健康状态仪表盘，智能台灯变为绿色（正常）或黄色（警告）
  - **听觉**：智能音箱播放健康建议，如"建议起身活动5分钟"
  - **辅助**：智能杯垫显示饮水量提醒，智能植物显示空气质量

- **功能**：
  - 监测久坐时间，超过1小时提醒休息
  - 监测环境光线，提醒调整台灯亮度保护眼睛
  - 监测饮水量，提醒及时补充水分
  - 监测室内温湿度，提供舒适度建议
  - 生成健康日报，显示当天的健康状态趋势

### 4. 任务管理
- **触发**：
  - 语音指令："桌面助手，添加任务"
  - 物理触发：点击智能按钮（双击）
  - 应用集成：从任务管理应用（如Todoist）同步

- **集群效果**：
  - **视觉**：智能显示器显示任务输入界面，输入完成后显示任务卡片，智能台灯变为绿色闪烁
  - **听觉**：智能音箱播放"任务添加成功"的提示音
  - **提醒**：任务到期前，智能音箱播放提醒，智能台灯变为黄色

- **功能**：
  - 添加任务，设置标题、描述、截止时间、优先级
  - 自动从邮件、聊天记录中提取任务
  - 跟踪任务进度，显示完成情况
  - 生成任务报告，显示本周完成的任务数量
  - 与日历应用集成，自动添加任务到日历

### 5. 智能会议助手
- **触发**：
  - 语音指令："桌面助手，准备会议"
  - 日历集成：会议开始前15分钟自动触发
  - 物理触发：点击智能按钮（三连击）

- **集群效果**：
  - **视觉**：智能显示器显示会议议程和参会人员，智能台灯变为紫色
  - **听觉**：智能音箱播放"会议即将开始"的提示音，播放参会人员名单
  - **环境**：智能传感器监测室内温度，自动调节舒适环境

- **功能**：
  - 自动从日历中获取会议信息，显示议程和参会人员
  - 提供会议倒计时，提醒会议开始
  - 会议期间，智能台灯变为蓝色，表示勿扰
  - 会议结束后，自动生成会议纪要，提取关键决策和行动项
  - 与团队协作工具（如Slack、Microsoft Teams）集成

### 6. 智能学习助手
- **触发**：
  - 语音指令："桌面助手，开启学习模式"
  - 物理触发：长按智能按钮（5秒）
  - 应用集成：从学习应用（如Notion）触发

- **集群效果**：
  - **视觉**：智能显示器显示学习计划和进度，智能台灯变为蓝色
  - **听觉**：智能音箱播放学习提示音，如"开始学习，加油！"
  - **交互**：智能按钮变为学习模式快捷键，可快速切换功能

- **功能**：
  - 显示学习计划，跟踪学习进度
  - 提供番茄工作法计时器，交替学习和休息
  - 自动整理学习笔记，分类存储
  - 提供知识点复习提醒，根据艾宾浩斯遗忘曲线
  - 生成学习报告，显示学习时间和效果

### 7. 多模态交互增强
- **语音交互**：
  - 支持自然语言指令，如"桌面助手，明天有什么任务？"
  - 支持上下文理解，可进行多轮对话
  - 支持个性化语音，可选择不同语音风格

- **物理交互**：
  - 智能按钮支持不同按压方式（短按、长按、双击、三连击）触发不同功能
  - 智能传感器支持手势识别，如挥手切换功能
  - 支持触控操作，在智能显示器上直接操作

- **视觉交互**：
  - 智能台灯支持多种光效模式，如呼吸灯、渐变灯、脉冲灯
  - 智能显示器支持卡片式界面，可滑动切换不同功能
  - 支持状态可视化，通过颜色和图标直观显示当前状态

- **听觉交互**：
  - 智能音箱支持多种提示音，如成功、警告、错误
  - 支持语音合成，播放个性化提醒
  - 支持环境音效，如白噪音、自然音、专注音乐

- **环境交互**：
  - 智能传感器监测环境变化，自动调整台灯、音箱等设备
  - 支持场景模式，如工作、学习、休息、娱乐
  - 支持位置感知，当用户离开桌面时自动进入节能模式


## 四、技术实现要点

### 1. 硬件集成
- **ESP32开发板**：作为各节点的核心控制器，支持Wi-Fi/BLE通信
- **传感器模块**：光线传感器（BH1750）、温湿度传感器（DHT22）、水位传感器、空气质量传感器（MQ-135）
- **执行器模块**：LED矩阵（MAX7219）、扬声器（3W）、小型显示屏（0.96英寸OLED）、继电器
- **电源模块**：USB电源（5V 2A）或电池供电

### 2. 软件架构

#### 中央终端软件
- **开发语言**：Python（快速开发）或Electron（跨平台桌面应用）
- **核心模块**：
  - **集群管理模块**：
    - 设备发现和连接管理
    - 节点状态监控
    - 配置管理
  - **语音识别模块**：
    - 使用Porcupine（本地唤醒词检测）
    - 使用Vosk（本地语音识别）
    - 语音指令解析和执行
  - **数据分析模块**：
    - 文件分类和整理
    - 用户行为分析
    - 环境数据处理
  - **可视化模块**：
    - 集群状态显示
    - 信息卡片展示
    - 数据可视化
  - **应用集成模块**：
    - 与日历、任务管理应用集成
    - 与操作系统通知系统集成

#### 节点固件
- **开发环境**：Arduino IDE或PlatformIO
- **核心模块**：
  - **通信模块**：
    - MQTT客户端
    - Wi-Fi连接管理
    - 消息处理和转发
  - **传感器模块**：
    - 传感器数据采集
    - 数据预处理和过滤

  - **执行器模块**：
    - LED矩阵控制

  - **反馈模块**：
    - 光效控制
    - 音效控制
    - 触觉反馈（可选）
  - **本地逻辑模块**：
    - 基本状态管理
    - 本地规则执行

### 3. 通信协议

#### MQTT主题结构
```
desktop_assistant/
  ├── node1/
  │   ├── commands    # 中央终端发送到节点的命令
  │   ├── status      # 节点发送到中央终端的状态
  │   └── data        # 节点发送到中央终端的数据
  ├── node2/
  │   ├── commands
  │   ├── status
  │   └── data
  └── all/
      └── commands    # 发送到所有节点的命令
```

#### 消息格式（JSON）

###### 命令消息（中央终端 → 节点）
```json
{
  "type": "SET_MODE",
  "data": {
    "mode": "focus",
    "params": {
      "duration": 1500,
      "color": "warm_yellow"
    }
  }
}
```

###### 状态消息（节点 → 中央终端）
```json
{
  "type": "STATUS_UPDATE",
  "data": {
    "node_id": "node1",
    "status": "online",
    "battery": 85,
    "signal_strength": -50
  }
}
```

###### 数据消息（节点 → 中央终端）
```json
{
  "type": "SENSOR_DATA",
  "data": {
    "node_id": "node1",
    "sensor_type": "temperature",
    "value": 25.5,
    "timestamp": "2026-02-12T10:30:00Z"
  }
}
```

### 4. AI模型

#### 语音识别
- **本地模型**：
  - Porcupine（唤醒词检测）：支持自定义唤醒词
  - Vosk（语音识别）：轻量级本地语音识别
- **云端模型**（可选）：
  - Google Speech-to-Text
  - 百度语音识别

#### 行为分析
- **专注度检测**：
  - 基于键盘鼠标活动频率的简单算法
  - 阈值判断：连续5分钟无活动视为不专注
- **文件分类**：
  - 基于文件扩展名的规则分类
  - 基于文件名关键词的简单文本匹配

#### 环境优化
- **光线调节**：
  - 规则引擎：根据时间和环境光线自动调节台灯亮度
  - 预设场景：工作、阅读、休息
- **温湿度控制**：
  - 舒适度指数计算
  - 基于阈值的提醒

### 5. 数据存储

#### 中央终端存储

## 七、技术风险与应对措施

### 技术风险
1. **通信可靠性**：Wi-Fi/BLE连接可能不稳定
2. **传感器精度**：低成本传感器的精度可能有限
3. **语音识别准确性**：环境噪音可能影响语音识别效果
4. **电源管理**：多个节点同时运行可能导致电源不足

### 应对措施
1. **通信可靠性**：实现重连机制和消息缓存
2. **传感器精度**：校准传感器，使用多个传感器数据融合
3. **语音识别准确性**：使用本地语音识别，减少环境噪音影响
4. **电源管理**：使用USB电源供电，优化节点功耗
