Background / 痛点

手机很强，但在现实场景里往往不够低摩擦：你很难在走路、开会、做事时频繁掏手机、解锁、切 App。 结果就是：遗漏、切换成本高、输出缺少陪伴与空间感。

  • 输入端：语音/照片/文件/事件经常“来不及记、记不全”。
  • 输出端：提醒都挤在屏幕里，缺少声/光/屏/动作带来的“存在感反馈”。
  • 执行端：不是“能建议”，而是要“能闭环”——从意图到完成。
Scenes / 场景脑暴（精选）
  • 生活：一句“我想专注” → 灯光进入 focus 场景 + 语音报 Top1 + 自动建 30min 番茄钟。
  • 厨房指导：拍一下食材 → 语音分步指导 + 屏幕 checklist（可选 AR）。
  • 工作质控（加分项）：会议结束“总结一下” → 输出一致点/分歧点/行动项，并可回放审计。
  • 医疗（严肃版）：按键采集 → 结构化记录 → 风险升级提醒 → 全链路可审计。
注：MVP 先做 2 个场景打穿；其余作为路线图。
Tech Route / 技术路线
  • 感知：VisionClaw 思路（手机/眼镜音频+低帧率视觉）→ Gemini Live 实时理解。
  • 执行：OpenClaw 作为工具中枢（消息/日程/搜索/自动化）。
  • 物理输出：Home Assistant / Matter / MQTT 作为总线，驱动灯/空调/插座等。
  • 闭环：事件→建议→确认→执行→回放（trace）→下次更准。
